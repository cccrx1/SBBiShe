# 实验指令文档

本文档按论文主线重新整理正式实验顺序，默认流程固定为：

1. 配置并激活环境
2. 整理数据集
3. 训练干净模型（benign baseline）
4. 在 `0.1`、`0.05`、`0.01` 三个投毒率下，分别训练 `BadNets / Blended / WaNet / Refool`
5. 对上述攻击结果统一使用 `REFINE` 训练净化模型
6. 评估 `BA` 和 `ASR_NoTarget`

脚本入口约定（已更新）：

- 推荐统一入口：
  - `scripts/train_benign.py`
  - `scripts/train_attack.py`
  - `scripts/train_refine.py`
  - `scripts/eval_refine.py`
- 兼容入口仍可用：
  - `scripts/train_gtsrb_*.py`
  - `scripts/train_refine_gtsrb.py`
  - `scripts/eval_refine_gtsrb.py`
- 兼容入口当前是薄包装，内部会转发到统一入口，后续维护以统一入口为准。

本文档的组织原则是：

- 同一阶段尽可能使用同一组参数
- 真正变化的参数尽量只保留：
  - `--attack`（或对应脚本）
  - `--poisoned-rate`
  - `--reflection-dir`（仅 Refool 需要）
- 做多投毒率实验时，要特别注意不要串用 checkpoint

## 3. 统一变量与公共参数

为了保证每个阶段相同参数尽可能一致，建议先在 shell 中统一定义下面这些变量：

```bash
cd /root/SBBiShe

REPO_ROOT=/root/SBBiShe
DATA_ROOT="$REPO_ROOT/datasets"
EXP_ROOT="$REPO_ROOT/experiments"
REFLECTION_DIR="$DATA_ROOT/refool_reflections"

DEVICE=GPU
GPU_ID=0
SEED=666
BATCH_SIZE=128
NUM_WORKERS=8
TEST_INTERVAL=10
SAVE_INTERVAL=10

ATTACK_EPOCHS=100
REFINE_EPOCHS=150

PR_LIST=(0.1 0.05 0.01)
```

统一约定如下：

- benign 训练与攻击训练尽量保持同一组训练参数：
  - `DEVICE / GPU_ID / BATCH_SIZE / NUM_WORKERS / SEED / TEST_INTERVAL / SAVE_INTERVAL`
- 攻击阶段统一使用 `ATTACK_EPOCHS=100`
- REFINE 阶段统一使用 `REFINE_EPOCHS=150`
- 评估阶段没有 `num_workers / test_interval / save_interval` 这些参数，但应保持：
  - `DEVICE / GPU_ID / BATCH_SIZE / SEED`

## 5. 阶段一：训练干净模型

先训练干净模型，作为后续对照基线。

```bash
python scripts/train_benign.py \
  --dataset gtsrb \
  --data-root "$DATA_ROOT" \
  --experiment-root "$EXP_ROOT" \
  --device "$DEVICE" \
  --gpu-id "$GPU_ID" \
  --epochs "$ATTACK_EPOCHS" \
  --batch-size "$BATCH_SIZE" \
  --num-workers "$NUM_WORKERS" \
  --seed "$SEED" \
  --test-interval "$TEST_INTERVAL" \
```

说明：

- 这里故意让 benign 阶段与攻击阶段尽可能共享同一组训练参数
- 这样后续做 `No Defense` 对照时更一致

## 9. 正式结果建议：尽量显式绑定 checkpoint

如果你只是严格按顺序跑一遍，且每个攻击/投毒率只保留最新结果，那么脚本自动推断通常够用。

但如果你要：

- 重复跑同一个投毒率
- 多次试验后只挑其中一次作为正式结果
- 避免不同投毒率或不同时间的结果串组

那么建议在 REFINE 训练和评估阶段显式绑定 checkpoint。

### 9.1 REFINE 训练示例

```bash
python scripts/train_refine.py \
  --dataset gtsrb \
  --attack badnets \
  --data-root "$DATA_ROOT" \
  --experiment-root "$EXP_ROOT" \
  --attack-checkpoint /root/SBBiShe/experiments/attacks/BadNets/<your_attack_dir>/ckpt_epoch_100.pth \
  --device "$DEVICE" \
  --gpu-id "$GPU_ID" \
  --epochs "$REFINE_EPOCHS" \
  --batch-size "$BATCH_SIZE" \
  --num-workers "$NUM_WORKERS" \
  --seed "$SEED" \
  --test-interval "$TEST_INTERVAL" \
  --save-interval "$SAVE_INTERVAL" \
  --poisoned-rate 0.1
```

### 9.2 REFINE 评估示例

```bash
python scripts/eval_refine.py \
  --dataset gtsrb \
  --attack badnets \
  --data-root "$DATA_ROOT" \
  --experiment-root "$EXP_ROOT" \
  --attack-checkpoint /root/SBBiShe/experiments/attacks/BadNets/<your_attack_dir>/ckpt_epoch_100.pth \
  --refine-checkpoint /root/SBBiShe/experiments/refine/BadNets/train/<your_refine_dir>/ckpt_epoch_150.pth \
  --arr-path /root/SBBiShe/experiments/refine/BadNets/train/<your_refine_dir>/label_shuffle.pth \
  --device "$DEVICE" \
  --gpu-id "$GPU_ID" \
  --batch-size "$BATCH_SIZE" \
  --seed "$SEED" \
  --poisoned-rate 0.1
```

## 10. 推荐的记录方式

建议你在实验记录里固定保留下面三层信息：

### 10.1 干净模型

- benign checkpoint 路径
- benign eval 结果

### 10.2 攻击模型

对每个攻击、每个投毒率，记录：

- 攻击名称
- `poisoned_rate`
- 对应 checkpoint 路径
- Attack only 结果

### 10.3 REFINE 结果

对每个攻击、每个投毒率，记录：

- 对应攻击 checkpoint
- 对应 REFINE checkpoint
- `label_shuffle.pth` 路径
- `BA`
- `ASR_NoTarget`

## 11. 建议的最终顺序总结

正式实验建议严格按下面顺序执行：

1. 激活环境
2. 整理 GTSRB 测试集
3. 检查 Refool 反射图目录
4. 训练干净模型
5. 在 `0.1` 下训练四种攻击
6. 在 `0.05` 下训练四种攻击
7. 在 `0.01` 下训练四种攻击
8. 在 `0.1` 下训练四种攻击对应的 REFINE
9. 在 `0.05` 下训练四种攻击对应的 REFINE
10. 在 `0.01` 下训练四种攻击对应的 REFINE
11. 在 `0.1` 下评估四种攻击对应的 REFINE
12. 在 `0.05` 下评估四种攻击对应的 REFINE
13. 在 `0.01` 下评估四种攻击对应的 REFINE

## 12. 一句话原则

同一阶段除攻击类型、投毒率和 Refool 反射图目录外，其余参数尽量保持一致；正式结果优先保证可追溯、不串组，再追求省事。
