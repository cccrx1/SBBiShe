# 实验指令文档

本文档是项目唯一的实验执行手册，按"流程阶段"组织，涵盖 GTSRB（论文主线）与 CUB-200 两个数据集。

## 1. 当前支持范围

**已支持：**

- 数据集：`gtsrb`（论文主线，43 类，32×32）、`cub200`（200 类，128×128）
- 攻击：`badnets`、`blended`、`wanet`、`refool`
- 防御：`REFINE`
- 分类器：`ResNet18`
- 预训练加载：分类器/攻击训练侧支持从本地 checkpoint 或 `torchvision://resnet18` URI 加载预训练权重做 warm start

**不支持（未承诺实现）：**

- PACS 等其他数据集
- 其他分类器或攻击

**代码可信度说明：**

- 分类器预训练加载（`core/attacks/base.py`）已处理 `module.`/`model.` 前缀剥离、形状不匹配跳过、`conv1` 的 7×7→3×3 center crop 适配，实测可用于 CUB-200 这类"类别数变化但主干可迁移"的场景。
- `REFINE` 侧的 `--pretrain` 语义与分类器完全不同：它加载的是 UNetLittle 权重，不是分类器 warm start。请勿将 `train_refine.py --pretrain` 误当成分类器提速手段使用。
- `torchvision://resnet18` URI 在 `base.py` 中有底层支持，但 `default_attack_schedule()` / `default_refine_schedule()` 会对其调用 `to_path()`，行为可能与预期不符。**本文档不将 torchvision URI 列为已验证的正式工作路径。** 如需预训练加速，推荐使用本地训练好的 benign checkpoint。

## 5. 推荐实验流程

**核心原则：先小闭环验证，再逐步展开正式实验。**

### 三阶段设计

| 阶段 | 目标 | 时间成本 |
|------|------|---------|
| Smoke Test | 验证数据、transform、路径、评估链路 | 极低（1 epoch） |
| Feasibility / Warm-start | 验证收敛趋势，确定是否可正式跑 | 中（5-20 epoch） |
| Formal Runs | 完整结果，可论文使用 | 高（100/150 epoch） |

### 5.1 阶段 0：Smoke Test（所有数据集通用）

**目标：** 验证数据、transform、产物路径、评估链路是否正确，不追求精度。

**仅跑 BadNets 链路：**

```bash
# ===== Attack Smoke Test =====
python scripts/train_attack.py \
  --dataset cub200 --attack badnets \
  --data-root datasets --experiment-root experiments \
  --gpu-id 0 --epochs 1 --disable-schedule \
  --batch-size 16 --num-workers 0 \
  --test-interval 1 --save-interval 1

# ===== REFINE Smoke Test =====
python scripts/train_refine.py \
  --dataset cub200 --attack badnets \
  --data-root datasets --experiment-root experiments \
  --gpu-id 0 --epochs 1 --disable-schedule \
  --batch-size 16 --num-workers 0 \
  --test-interval 1 --save-interval 1

# ===== Eval Smoke Test =====
python scripts/eval_refine.py \
  --dataset cub200 --attack badnets \
  --data-root datasets --experiment-root experiments \
  --gpu-id 0 --batch-size 16
```

Smoke Test 关注点：

- 数据能否正常加载（无 crash、无路径错误）
- transform 是否正确（图像尺寸、归一化范围合理）
- 产物目录是否生成（checkpoint、日志）
- 评估链路是否完整（BA / ASR_NoTarget 都有输出）

如果 smoke test 失败，立刻停在当前阶段排查，不进入正式训练。

### 5.2 阶段 1：Feasibility / Warm-start

**目标：** 在 smoke test 通过后，验证 benign 基线与攻击训练的收敛趋势。

#### 5.2.1 Benign Warm-up（可选但推荐）

先用 benign 基线验证训练链路是否正常，同时积累可复用的 warm start checkpoint：

```bash
python scripts/train_benign.py \
  --dataset cub200 --data-root datasets --experiment-root experiments \
  --gpu-id 0 --epochs 5 --disable-schedule \
  --batch-size 32 --num-workers 0 \
  --test-interval 1 --save-interval 1

# 服务器
python /root/SBBiShe/scripts/train_benign.py \
  --dataset cub200 --data-root /root/SBBiShe/datasets \
  --experiment-root /root/SBBiShe/experiments \
  --gpu-id 0 --epochs 5 --disable-schedule \
  --batch-size 32 --num-workers 8 \
  --test-interval 1 --save-interval 1
```

如果 benign 在 5 epoch 内 accuracy 有明显上升趋势，继续下一步。

#### 5.2.2 攻击可行性验证

如果 benign warm-up 有合理 checkpoint，可将其作为预训练权重加速攻击训练收敛：

```bash
# 假设 benign checkpoint 在 experiments/benign/cub200/
PRETRAIN_CKPT=/root/SBBiShe/experiments/benign/cub200/ckpt_epoch_5.pth

python scripts/train_attack.py \
  --dataset cub200 --attack badnets \
  --data-root /root/SBBiShe/datasets \
  --experiment-root /root/SBBiShe/experiments \
  --gpu-id 0 --epochs 20 --disable-schedule \
  --batch-size 32 --num-workers 8 \
  --test-interval 1 --save-interval 1 \
  --pretrain "$PRETRAIN_CKPT"
```

关注点：

- 攻击训练 loss 是否正常下降
- Clean accuracy 是否维持
- Poisoned accuracy / ASR 是否有后门效果

如果 20 epoch 后趋势合理，再进入正式实验。如果 benign 或 badnets 单攻击都没有收敛趋势，**先停在这一阶段排查**，不展开全量四攻击。

### 5.3 阶段 2：Formal Runs

**目标：** 产出论文可用结果。

**推荐顺序：** 先做 `badnets → REFINE → eval`，通过后再扩展到 `blended`、`wanet`、`refool`。不要一次性把所有攻击全训完才评估。

**每个攻击的扩展顺序：** badnets → blended → wanet → refool

**显式绑定 checkpoint（强烈建议）：**

做正式实验时，每次训练和评估应显式传入 checkpoint 路径，避免多组实验产物串组：

```bash
# REFINE 训练（显式绑定攻击模型）
python scripts/train_refine.py \
  --dataset cub200 --attack badnets \
  --data-root /root/SBBiShe/datasets \
  --experiment-root /root/SBBiShe/experiments \
  --attack-checkpoint /root/SBBiShe/experiments/attacks/BadNets/cub200_badnets_pr0.05/ckpt_epoch_100.pth \
  --gpu-id 0 --epochs 150 \
  --batch-size 32 --num-workers 8 \
  --test-interval 10 --save-interval 10 \
  --poisoned-rate 0.05

# REFINE 评估（显式绑定所有产物）
python scripts/eval_refine.py \
  --dataset cub200 --attack badnets \
  --data-root /root/SBBiShe/datasets \
  --experiment-root /root/SBBiShe/experiments \
  --attack-checkpoint /root/SBBiShe/experiments/attacks/BadNets/cub200_badnets_pr0.05/ckpt_epoch_100.pth \
  --refine-checkpoint /root/SBBiShe/experiments/refine/BadNets/cub200_refine_badnets_train_pr0.05/ckpt_epoch_150.pth \
  --arr-path /root/SBBiShe/experiments/refine/BadNets/cub200_refine_badnets_train_pr0.05/label_shuffle.pth \
  --gpu-id 0 --batch-size 32 \
  --poisoned-rate 0.05
```

## 8. CUB-200 Transform 与触发器插入位

### 8.1 Transform 链路

`scripts/_common.py` 中 `make_cub200_transforms()` 当前定义：

- **train：** `ToPILImage → Resize(147×147) → RandomCrop(128×128) → RandomHorizontalFlip → ToTensor → Normalize`
- **test：** `ToPILImage → Resize(147×147) → CenterCrop(128×128) → ToTensor → Normalize`

### 8.2 触发器插入位

由于 trigger / warp 操作依赖真实像素值，必须在 resize/crop/flip 之后、`ToTensor/Normalize` 之前注入：

| 攻击 | train 插入位 | test 插入位 |
|------|------------|-----------|
| BadNets / Blended / WaNet | 4 | 3 |
| Refool | 0（pre/post 切分设计，勿改动） | 0 |

> **如果后续改动 `make_cub200_transforms()`，必须同步复核 `scripts/_common.py` 中的 `poisoned_transform_indices()` 函数**，否则容易引入尺寸或顺序错位问题。

## 9. 默认攻击配置

## 11. 结果绑定与排查规则

### 11.1 显式绑定规则

做多投毒率实验时，**强烈建议**显式传入以下参数：

```bash
--poisoned-rate
--attack-checkpoint
--refine-checkpoint
--arr-path
```

原因：`scripts/_common.py` 会自动推断最新 checkpoint，如果目录里同时存在多组投毒率，容易误绑到别的实验产物。

### 11.2 异常排查顺序

1. 是否加载了与当前 `--poisoned-rate` 对应的攻击 checkpoint
2. 评估时是否显式绑定了 `--refine-checkpoint` 和 `--arr-path`
3. Refool 是否使用了正确的 `--reflection-dir`
4. WaNet 是否使用了与训练时一致的 `grid_k` / grid 文件
5. 比较 `ASR_NoTarget` 时，是否在同一攻击、同一投毒率下做前后对照
6. 如果 REFINE 后 ASR 不降，先确认该次 REFINE 是否确实用 clean dataset 训练

### 11.3 额外注意

- WaNet 依赖 `identity_grid` 和 `noise_grid`，相关文件会保留在结果目录中
- Refool 对反射图内容较敏感，跨机器复现时需确认目录内容一致
- CUB-200 若出现触发器或 warp 尺寸异常，优先检查 `make_cub200_transforms()` 与 `poisoned_transform_indices()` 是否仍保持同步
- 如果环境里设置了 `OMP_NUM_THREADS` 或 `MKL_NUM_THREADS`，确保它们是正整数；空值或 `0` 可能触发 `libgomp: Invalid value for environment variable`
