# 项目索引文档

## 1. 项目定位

本仓库是面向本科毕业论文的最小实验版本，主线固定为：

- 数据集：GTSRB
- 分类模型：ResNet18
- 攻击方法：BadNets、Blended、WaNet、Refool
- 防御方法：REFINE

目标不是保留通用 backdoor toolbox，而是支持一条清晰、可复现实验链路：

1. 训练 benign 基线
2. 训练 4 种攻击模型
3. 针对每种攻击训练 REFINE
4. 评估 BA 和 ASR_NoTarget

## 3. 核心模块

- `core/__init__.py`

- `core/attacks/base.py`
  - 攻击统一基类，想理解攻击训练范式应先读这里。
- `core/attacks/`
  - 当前只保留 `BadNets`、`Blended`、`WaNet`、`Refool`。
- `core/defenses/base.py`
  - 防御统一基类。
- `core/defenses/REFINE.py`
  - REFINE 主实现，包含 UNet 训练、预处理与推理逻辑。

  - GTSRB 主分类器使用的 ResNet18。
- `core/models/unet.py`
  - REFINE 使用的 `UNetLittle`。
- `scripts/`
  - 真实实验入口，不是示例脚本集合。

## 5. 论文实验中的关键约定

### 数据集组织

GTSRB 使用 `torchvision.datasets.DatasetFolder` 读取，要求：

- `datasets/GTSRB/train/00000 ... 00042/`
- `datasets/GTSRB/testset/00000 ... 00042/`

原始 GTSRB 测试集通常是扁平目录，必须先运行：

```bash
python scripts/prepare_gtsrb_testset.py --data-root datasets
```

### REFINE 训练口径

这是当前论文实验最重要的约定：

- 攻击模型正常用 poisoned dataset 训练
- REFINE 训练时使用干净的 `trainset/testset`
- REFINE 评估时再对 poisoned testset 计算 `ASR_NoTarget`

这样做的原因是：`REFINE.py` 内部使用攻击模型预测作为伪标签，如果直接拿 poisoned dataset 训练，容易把后门行为一起保留下来。

## 6. 典型调用路径

### 攻击训练

1. 读取 GTSRB clean train/test
2. 构造 ResNet18
3. 实例化攻击类
4. 调用 `train()`
5. 调用 `test()`
6. 保存攻击 checkpoint

### REFINE 训练

1. 读取 GTSRB clean train/test
2. 加载已训练好的攻击模型 checkpoint
3. 实例化 `REFINE(UNetLittle, attacked_model)`
4. 用 clean train/test 调用 `train_unet()`
5. 保存 UNet checkpoint 与 `label_shuffle.pth`

### REFINE 评估

1. 加载攻击模型
2. 加载 REFINE checkpoint 与 `label_shuffle.pth`
3. 在 clean testset 上算 BA
4. 在 poisoned testset 上算 ASR_NoTarget

## 7. 主要脚本

- `scripts/prepare_gtsrb_testset.py`
  - 整理 GTSRB 原始测试集目录。
- `scripts/bootstrap_datasets.py`
  - 在无现成数据集时自动下载并整理 GTSRB/CUB200 到项目可用格式。
- `scripts/prepare_cub200_dataset.py`
  - 把 CUB-200-2011 原始目录整理成 `DatasetFolder` 结构。
- `scripts/train_benign.py`
  - 通用 benign 训练入口（支持 `--dataset`）。
- `scripts/train_attack.py`
  - 通用攻击训练入口（支持 `--dataset` + `--attack`）。
- `scripts/train_refine.py`
  - 通用 REFINE 训练入口（支持 `--dataset` + `--attack`）。
- `scripts/eval_refine.py`
  - 通用 REFINE 评估入口（支持 `--dataset` + `--attack`）。
- `scripts/collect_final_results.py`
  - 汇总 `experiments/refine/*/eval/*/results.txt` 到单文件，并支持论文宽表导出。
- `scripts/train_gtsrb_benign.py` / `scripts/train_gtsrb_badnets.py` / `scripts/train_gtsrb_blended.py` / `scripts/train_gtsrb_wanet.py` / `scripts/train_gtsrb_refool.py`
  - GTSRB 兼容入口，内部转发到通用脚本。
- `scripts/train_refine_gtsrb.py` / `scripts/eval_refine_gtsrb.py`
  - GTSRB 兼容入口，内部转发到通用脚本。

## 8. AI 使用建议

- 想理解攻击共性：先读 `core/attacks/base.py`
- 想理解 REFINE 逻辑：重点读 `core/defenses/REFINE.py`
- 想看真实实验入口：读 `scripts/`
- 想核对论文命令：读 `实验指令文档.md`
- 想核对结果记录口径：读 `结果文档.md`

## 9. 已知特殊点

- `ASR_NoTarget` 必须基于 poisoned testset 的原始 clean 标签统计，不能用投毒后的目标标签替代。
- WaNet 依赖 `identity_grid` 和 `noise_grid`，结果目录下会保留对应文件。
- Refool 必须提供单独的反射图目录，例如 `datasets/refool_reflections/`。
- 做多投毒率实验时，推荐显式传入：
  - `--poisoned-rate`
  - `--attack-checkpoint`
  - `--refine-checkpoint`
  - `--arr-path`
