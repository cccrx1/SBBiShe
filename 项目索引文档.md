# 项目索引文档

## 1. 项目定位

本仓库是面向本科毕业论文的最小实验版本。论文主线固定为：

- 数据集：GTSRB（论文主线）
- 分类模型：ResNet18
- 攻击方法：BadNets、Blended、WaNet、Refool
- 防御方法：REFINE

当前统一入口另外支持：

- CUB-200 单域分类实验（消融/验证用）

**不支持：** PACS 等其他数据集。目标不是保留通用 backdoor toolbox，而是围绕一条清晰、可复现的实验链路开展工作。

## 4. 数据集配置

`scripts/_common.py` 中的 `DATASET_SPECS` 定义了当前支持的数据集：

| 键 | num_classes | image_size | train_subdir | test_subdir |
|----|-----------|-----------|-------------|------------|
| `gtsrb` | 43 | 32 | `GTSRB/train` | `GTSRB/testset` |
| `cub200` | 200 | 128 | `CUB200/train` | `CUB200/test` |

### GTSRB 测试集必须先整理

原始 GTSRB 测试集通常是扁平目录，`load_gtsrb_datasets()` 会检查并报错。必须先执行：

```bash
python scripts/prepare_gtsrb_testset.py --data-root datasets
```

### CUB-200 Transform 与攻击插入位

`make_cub200_transforms()` 定义的 transform 链路：

- **train：** `ToPILImage → Resize(147×147) → RandomCrop(128×128) → RandomHorizontalFlip → ToTensor → Normalize`
- **test：** `ToPILImage → Resize(147×147) → CenterCrop(128×128) → ToTensor → Normalize`

触发器 / warp 操作依赖真实像素值，必须插在 crop/flip 之后、`ToTensor/Normalize` 之前：

| 攻击 | train 插入位 | test 插入位 |
|------|------------|-----------|
| BadNets / Blended / WaNet | 4 | 3 |
| Refool | 0（pre/post 切分设计） | 0 |

> **如果后续改动 `make_cub200_transforms()`，必须同步复核 `poisoned_transform_indices()`，否则容易引入尺寸或顺序错位问题。**

### Refool 反射图

Refool 依赖单独的反射图目录：

- `datasets/refool_reflections`（本地）
- `/root/SBBiShe/datasets/refool_reflections`（服务器）

`load_reflection_images()` 会检查目录是否存在及图像是否可读，不满足则报错。

## 5. 预训练加载能力与限制

### 5.1 分类器 / 攻击训练侧（`core/attacks/base.py`）

支持通过 `--pretrain` 传入本地 checkpoint 路径，加载到 ResNet18 主干。

处理逻辑（`_load_pretrain_robust`）：

1. 支持 `torchvision://resnet18` URI 和本地 `.pth` 文件路径
2. 自动剥离 `module.` / `model.` 前缀（兼容 DataParallel / HuggingFace 保存格式）
3. 跳过形状不匹配的层（source ≠ target）
4. 对 `conv1` 的 7×7 → 3×3 形状差异做 center crop 适配（用于 ImageNet 预训练迁移到下游任务）
5. 记录加载统计：loaded / adapted / skipped_by_shape / skipped_missing

该机制已实测可用于 CUB-200 这类"类别数变化但主干可迁移"的场景。**推荐预训练加速路径：使用本地训练好的 benign checkpoint，而非 torchvision URI。**

### 5.2 REFINE 侧（`core/defenses/REFINE.py` + `default_refine_schedule`）

`REFINE` 的 `--pretrain` 语义与分类器完全不同：它加载的是 `UNetLittle` 权重，不是分类器 checkpoint。

**请勿将 `train_refine.py --pretrain` 误当成分类器提速手段。** 当前文档不将 `torchvision://resnet18` 列为已验证的正式工作路径。

## 6. 统一实验口径

### 6.1 攻击训练

攻击模型正常使用 poisoned dataset 训练。

### 6.2 REFINE 训练

**必须使用 clean trainset/testset**，不能直接使用 poisoned dataset。

原因：`REFINE.py` 内部使用攻击模型预测作为伪标签，若输入本身是 poisoned dataset，训练目标可能会保留后门预测。当前论文主线正是通过 clean-data training 检验 REFINE 是否真正削弱触发器效果。

`scripts/train_refine.py` 已按该口径实现：读取 clean trainset/testset → 加载攻击模型 checkpoint → 实例化 REFINE → 调用 `defense.train_unet()`。

### 6.3 REFINE 评估

- **BA**：在 clean testset 上评估模型分类准确率
- **ASR_NoTarget**：在 poisoned testset 上评估，但统计时使用原始 clean 标签，并忽略原始标签本来就等于目标类的样本

必须固定以下几点：

- 不能把 poisoned label 直接当评估真值
- 不能把"目标标签命中率"误写成普通分类准确率
- 结果分析时应始终与同攻击、同投毒率、同 checkpoint 组进行对照

`scripts/eval_refine.py` 和 `_common.py:manual_refine_eval()` 已按该口径实现。

## 8. 产物目录与结果汇总

### 8.1 产物目录

```
experiments/
|-- benign/                       # benign 基线
|-- attacks/
|   |-- BadNets/                  # BadNets 训练产物
|   |-- Blended/                  # Blended 训练产物

```

### 8.2 关键产物说明

| 产物 | 说明 |
|------|------|
| `ckpt_epoch_*.pth` | 训练 checkpoint |
| `label_shuffle.pth` | REFINE 标签随机打乱映射 |
| `log.txt` | REFINE 训练日志 |
| `results.txt` | 单次评估结果（BA / ASR_NoTarget） |

### 8.3 结果汇总

```bash
python scripts/collect_final_results.py \
  --experiment-root experiments --latest-only \
  --output experiments/final_summary/final_results.csv \
  --paper-output experiments/final_summary/final_results_paper.md \
  --paper-format md
```

支持：

- 按 `--dataset` 过滤
- `--latest-only` 只取每个攻击+投毒率组合的最新结果
- CSV / JSON / Markdown 输出
- 论文宽表格式（Attack × Poisoned Rate）

## 9. 命令与参数约定

### 9.1 攻击名称

`--attack` 统一使用小写：`badnets`、`blended`、`wanet`、`refool`

内部 canonical 映射：badnets → BadNets、blended → Blended、wanet → WaNet、refool → Refool

### 9.2 产物绑定（多投毒率实验必读）

做多投毒率实验时，`scripts/_common.py` 会自动推断最新 checkpoint。如果目录里同时存在多组投毒率，容易误绑到别的实验产物。

**强烈建议显式传入：**

```bash
--poisoned-rate
--attack-checkpoint
--refine-checkpoint
--arr-path
```

### 9.3 常用参数一览

**通用参数：** `--dataset`、`--data-root`、`--experiment-root`、`--gpu-id`、`--device`、`--batch-size`、`--num-workers`、`--seed`

**训练参数：** `--epochs`、`--lr`、`--gamma`、`--schedule`、`--disable-schedule`

**攻击参数：** `--poisoned-rate`、`--y-target`、`--trigger-size`、`--blended-alpha`、`--wanet-grid-k`、`--reflection-dir`、`--reflection-limit`

**产物绑定：** `--attack-checkpoint`、`--refine-checkpoint`、`--arr-path`

## 10. 结果异常排查顺序

1. 是否加载了与当前 `--poisoned-rate` 对应的攻击 checkpoint
2. 评估时是否显式绑定了 `--refine-checkpoint` 和 `--arr-path`
3. Refool 是否使用了正确的 `--reflection-dir`
4. WaNet 是否使用了与训练时一致的 `grid_k` / grid 文件
5. 比较 `ASR_NoTarget` 时，是否在同一攻击、同一投毒率下做前后对照
6. 如果 REFINE 后 ASR 不降，先确认该次 REFINE 是否确实用 clean dataset 训练

**额外注意：**

- WaNet 依赖 `identity_grid` 和 `noise_grid`，相关文件会保留在结果目录中
- Refool 对反射图内容较敏感，跨机器复现时需确认目录内容一致
- CUB-200 若出现触发器或 warp 尺寸异常，优先检查 `make_cub200_transforms()` 与 `poisoned_transform_indices()` 是否仍保持同步
- 如果环境里设置了 `OMP_NUM_THREADS` 或 `MKL_NUM_THREADS`，确保它们是正整数
