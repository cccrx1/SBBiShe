## 2. 结果指标口径

### BA

`BA` 表示 benign accuracy，即在 clean testset 上的分类准确率。

### ASR_NoTarget

`ASR_NoTarget` 表示在 poisoned testset 上，仅统计原始标签不等于目标标签的样本，计算被攻击到目标标签的比例。

这个口径必须固定，不能把 poisoned label 直接当作评估真值。

## 3. 当前论文中的防御口径

REFINE 当前推荐实验口径为：

1. 攻击模型使用 poisoned dataset 训练
2. REFINE 使用 clean trainset/testset 训练
3. 最终在 clean testset 上测 BA
4. 最终在 poisoned testset 上测 ASR_NoTarget

这样才能检验 REFINE 是否真的削弱了触发器效果。

## 4. 结果异常时的检查顺序

1. 是否加载了与当前 `--poisoned-rate` 对应的攻击 checkpoint
2. 评估时是否显式绑定了 `--refine-checkpoint` 和 `--arr-path`
3. Refool 是否使用了正确的 `--reflection-dir`
4. WaNet 是否使用了与训练时一致的 grid 配置
5. 比较 `ASR_NoTarget` 时，是否在同一攻击、同一投毒率下做前后对照
6. 如果 REFINE 后 ASR 不降，先确认该次 REFINE 是否是用 clean dataset 训练
