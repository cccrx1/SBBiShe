# 结果文档

## 一、结果目录映射

- `experiments/benign/`
  - benign 基线模型与日志
- `experiments/attacks/BadNets/`
- `experiments/attacks/Blended/`
- `experiments/attacks/WaNet/`
  - 包含 `grids/`（identity_grid / noise_grid）
- `experiments/attacks/Refool/`
- `experiments/refine/<Attack>/train/`
  - REFINE 训练产物：`ckpt_epoch_*.pth`、`label_shuffle.pth`、`log.txt`
- `experiments/refine/<Attack>/eval/`
  - REFINE 评估结果：`results.txt`

## 二、应优先保留的文件

1. 每个攻击的最终 checkpoint
2. 每个攻击对应的 REFINE checkpoint
3. 每个攻击对应的 `label_shuffle.pth`
4. `log.txt` 与 `results.txt`
5. WaNet 的 grid 文件

## 三、指标定义

- BA（Benign Accuracy）
  - 在干净测试集上的准确率
- ASR_NoTarget
  - 在 poisoned test 样本中，仅统计原始标签不等于目标标签的样本，计算攻击成功率

## 四、结果口径检查

当出现异常结果时，按以下顺序排查：

1. 是否使用了对应投毒率的攻击 checkpoint
2. 评估是否显式绑定了 `refine-checkpoint` 和 `arr-path`
3. 结果文件中是否记录了 `poisoned_rate`、`y_target`
4. 同一攻击下是否做了“防御前 vs 防御后”同口径对比
